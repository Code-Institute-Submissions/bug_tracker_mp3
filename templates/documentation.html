Introduction

This is a webapp aimed to work as a tool to help the management of Projects and Tickets within a Team. It was designed to be simple and easy to use.

Regular User

Regular Users have access only to the following essential functionalities on the App:

  • Edit own user Location
  • Change Password
  • Create Tickets
  • Edit, Close and Reopen Tickets

Back to the Index

Admin/Manager User

The Admin/Manager has access to all functionalities of the Regular User and the following extra ones:

  • Search, Edit another Users (This include upgrading another user to the Manager Category)
  • Change own's and another users' Password
  • Create Projects and select who will be working on it
  • Archive or Delete Projects with its Tickets

Back to the Index

User Registration

Every user when registered at first is automatically placed into the Regular category for security reasons. Only the Manager has the power to include this new user in a project or transform this new user into another Manager.
Also by default after a user is created he is required to login to enter the App.

Back to the Index

User Login

The Login form validates user and encripted password. There is an extra option for demonstration purposes where a user after creating its own login (that will be on the Regular Category) can also login as an "Admin" to include the user in a Project and check how the Manager sees the system.

When logging in with a regular user, there will be a greeting and a warning informing to contact the manager to be included in a Project. This way the system is safe of anybody creating users and deleting/creating data from the Database.

Back to the Index

Create Project

This option is only for Admin/Manager. When opening this form, the Manager will already be included on the project and can include additional users to complete the Team working on the Project.

Back to the Index

Create Ticket

This option is available to every user that has a Project assigned. The ticket created will be automatically assigned to the user that is creating it. There is the Edit option on the Dashboard later if there is the necessity to assign it to anybody else from the team.

Back to the Index

Dashboard Projects (Admin/Manager)

As a Manager all Projects will have the option to be Archived (It will also close all tickets related to it) and the option to Delete (It will also delete all tickets related to it). The Manager is the only one able to Archive or Delete Projects.
The Dashboard also contains the weather information of the user location and Total of Projects and its Tickets.

Back to the Index

Dashboard Projects (Regular User)

Regular Users are able to see the information of the Projects they are assigned, but they are not able to take any action as Archive and Delete are only for Admin/Manager Users.
The Dashboard also contains the weather information of the user location and Total of Projects and its Tickets.

Back to the Index

Dashboard Tickets

The tickets are shown to all users assigned to the related Projects. They are classified in 3 different colours:
Blue: Tickets assigned to another member of the Team.
Green: Tickets assigned to own user.
Grey: Tickets closed.

Back to the Index

Edit Tickets

Tickets can be edited through the "Edit" button on the Dashboard part for them. At this place the ticket can be assigned to another user if necessary.

Back to the Index

Manage User

As a Admin/Manager, it will be able to search for another users and select them to Edit or change Password.

As a Regular user, the search functionality is not available and it is possible to Edit or change Password of its own User.

Back to the Index

Edit User

As a Admin/Manager, it will be able change user Location and upgrade the user to "Manager" if necessary.

As a Regular user, it can only change the user Location to have a weather update of the place they are working.

Back to the Index

Change Password

Regular Users are able to change its own password. If an user is not able to remember his password, an Admin/Manager is able to reset it.
